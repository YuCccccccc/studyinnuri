
Swagger과 Restdocs

Swagger는 마치 Postman처럼(직접 요청하듯이) API를 테스트해 볼 수 있는 화면을 제공하여 동작 테스트하는 용도에 특화
Swagger는 문서화도 되고 테스트도 가능

Swagger 단점
로직에 애노테이션을 통해 명세를 작성하게 되는데 지속적으로 사용하게 된다면 
명세를 위한 코드들이 많이 붙게되어 전체적으로 가독성이 떨어진다.
테스트 기반이 아니기에 문서가 100% 정확하다고 확신할 수 없다.
모든 오류에 대한 여러 가지 응답을 문서화할 수 없다.

Restdocs
Asciidoctor를 이용해서 HTML 등등 다양한 포맷으로 문서를 자동으로 출력
RestDocs의 가장 큰 장점은 테스트 코드 기반으로 문서를 작성
API Spec과 문서화를 위한 테스트 코드가 일치하지 않으면 테스트 빌드를 실패하게 되어 테스트 코드로 검증된 문서를 보장


	Spring Rest Docs	                Swagger
장점	제품코드에 영향 없다.	    API 를 테스트 해 볼수 있는 화면을 제공한다.
    테스트가 성공해야 문서작성된다.	        적용하기 쉽다.
단점	적용하기 어렵다.	        제품코드에 어노테이션 추가해야한다.
    제품코드와 동기화가 안될수 있다.
